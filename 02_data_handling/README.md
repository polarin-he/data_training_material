# POLARIN Data Curation Training Material
## Repository Content
This folder provides guidance on combining, formatting, transforming, preparing data to adhere to the FAIR data principles. It emphasizes the importance of maintaining data accessibility and usability over time.

### Key Topics
- **FAIR data principles**
- **Internationally accepted FAIR (meta-)data standards and file structures**
- **Structuring datasets for interoperability and reusability**

### Structure of repository
1. **Learning Modules**                                          
2. **Case Studies**
3. **Additional Resources**

## Contributing to this training material repository
If u want to contribute to the contents of this repository, open an issue here on GitHub, make changes to a new branch and make a pull request, or send an email to the content curator Daan Kivits: daan.kivits@sios-svalbard.org.

## Note on the use of AI in these Jupyter Notebooks
The contents Jupyter Notebooks in this repository have partially been made using the GPT-4o Language Model developed by OpenAI, but the output is proofread by a gruop of data management experts connected to the POLARIN project. Learn more about the GPT-4o Language Model [here](https://openai.com/index/hello-gpt-4o/).

## Note on FAIRness of these Jupyter Notebooks
The Jupyter Notebooks in this repository are designed to be FAIR themselves. In designing these Jupyter Notebooks, the rules outlined in the "[FAIR for Jupyter Notebooks: A Practical Guide](https://ardc.edu.au/resources/fair-for-jupyter-notebooks-a-practical-guide/)" were taken into account.
